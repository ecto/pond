# PAD Quick Start

Learn how to use PAD (Pond Application Dashboard) for teleoperation and monitoring.

## What is PAD?

PAD is your primary interface for interacting with Pond robots. It provides:

- **Real-time 3D visualization** of the robot and environment
- **Teleoperation controls** for manual robot control
- **Sensor monitoring** with live data feeds
- **System diagnostics** for health and debugging
- **Configuration management** for tuning parameters

## Installation

PAD is included with Pond. Build it with:

```bash
cargo build -p pad --release
```

## Running the System

### 1. Start the Simulation Server

First, launch the simulation server that PAD will connect to:

```bash
cargo run --bin server --features server -p sim
```

You should see:
```
üê∏ Pond Sim Server listening on 0.0.0.0:8080
```

### 2. Launch PAD

In a new terminal, start PAD:

```bash
cargo run -p pad
```

Or connect to a remote server:

```bash
cargo run -p pad -- --server ws://192.168.1.100:8080
```

## Command-Line Options

```bash
pad [OPTIONS]

Options:
  -s, --server <SERVER>    Simulation server address [default: ws://localhost:8080]
      --fullscreen         Start in fullscreen mode
  -h, --help              Print help
```

## Using with Real Hardware

PAD works the same whether connected to simulation or real hardware:

1. Ensure your robot is running a compatible WebSocket server
2. Connect PAD with `--server` pointing to the robot's IP
3. The protocol is identical, so switching is seamless

## Standalone Sim View

You can also run just the 3D viewer without the full PAD interface:

```bash
cargo run -p sim-view -- --server ws://localhost:8080
```

This is useful for:
- Lightweight visualization
- Multiple simultaneous views
- Embedding in other applications
- Remote monitoring without control access

## Troubleshooting

### "Connection refused"

- Check that the sim server is running: `cargo run --bin server --features server -p sim`
- Verify the server address matches (default: `ws://localhost:8080`)
- Check firewall settings if connecting remotely

### "Compilation error: bevy_egui"

```bash
cargo clean
cargo build -p pad
```

### Black screen or no robot visible

- Switch to Sim View tab (press `1`)
- Check Settings tab to ensure "Follow robot camera" is enabled
- Verify the sim server is sending state updates (check server terminal)
